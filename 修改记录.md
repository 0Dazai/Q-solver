# 修改记录

## 2026-06-21：同步本地修改版到 GitHub fork

本次变更将本地修改后的 Q-Solver 源码同步到 `0Dazai/Q-solver` 仓库，用于保留当前可运行版本和后续继续维护。

## 主要修改

1. 重构桌面应用后端结构
   - 将原先集中在根目录 `app.go` 的逻辑拆分为 `app/` 目录下多个文件。
   - 新增或同步 `app/app.go`、`app/llm.go`、`app/settings.go`、`app/solve.go`、`app/window.go`、`app/screen.go`、`app/shortcut.go`、`app/resume.go`、`app/misc.go`。
   - 目的：让模型、设置、解题、窗口、截图、快捷键等入口更清晰。

2. 增加模型 Provider 和豆包 / 火山方舟兼容支持
   - 配置中增加 `provider`、`baseURL`、`model`、`prompt` 等字段。
   - 支持 OpenAI 兼容接口、DeepSeek 文本模型、豆包 / 火山方舟视觉模型。
   - 示例配置见 `config/config.example.json`。

3. 增强 LLM 请求能力边界
   - DeepSeek 保留纯文本能力。
   - 图片或 PDF 请求会在发送前进行能力判断，避免把 `image_url` 发给不支持视觉输入的模型。
   - 新增 `pkg/llm/openai_test.go` 覆盖关键 LLM 请求行为。

4. 支持手动填写模型 ID / 接入点 ID
   - 前端模型设置支持手动输入模型 ID 或火山方舟 `ep-...` 接入点 ID。
   - 适配火山方舟控制台中模型权限、地域、项目不一致时的真实使用场景。

5. 增加用户附加要求
   - 设置页支持填写“附加要求”。
   - 附加要求会注入解题提示，用于控制编程题默认语言等偏好。
   - 避免把默认语言硬编码为 Python、Java 或其他固定语言。

6. 调整窗口、最小化和隐身能力边界
   - 普通最小化保持任务栏可恢复。
   - F9 隐身能力继续与 Windows display affinity 相关逻辑保持独立。
   - 相关代码位于 `pkg/state`、`pkg/platform`、`pkg/shortcut` 和 `app/window.go`。

7. 重构前端状态和组件
   - 新增 `frontend/src/stores/settings.js`、`frontend/src/stores/solution.js`、`frontend/src/stores/ui.js`。
   - 新增 `frontend/src/services/api.js`、`events.js`、`theme.js`。
   - 新增 `DomainSelector`、`ScreenshotDock`、`SolveView`、`ThemeToggle`、`ThinkingBlock` 等组件。
   - 设置页恢复分类 prompt 展示，并保留“附加要求”配置入口。

8. 新增工具化截图模块
   - 新增 `pkg/tools/registry.go`、`pkg/tools/screenshot.go`、`pkg/tools/types.go`。
   - 为后续工具化截图和模型调用扩展提供统一入口。

9. 清理旧结构和生成物
   - 删除旧根目录 `app.go`。
   - 删除旧的 live/audio/Claude/Gemini 等未保留模块。
   - 删除 `.DS_Store`、`.idea` 等不应继续跟踪的本地文件。
   - `.gitignore` 更新为忽略构建产物、本地配置、依赖目录和缓存。
